# Product

## Register

product

## Users

Orbit is for a single person managing the practical threads of daily life on a Mac: habits, ideas, tasks, workflows, and relationships. The user expects a calm desktop tool that stays available throughout the day, rewards keyboard fluency, and never requires an account or network connection.

## Product Purpose

Orbit is a local-first personal operating system. It brings habit tracking, connected notes, visual task workflows, and lightweight personal CRM into one coherent application. Success means the user can capture something quickly, understand how their work and ideas connect, and trust that all information remains private, portable, and available offline.

## Brand Personality

Calm, focused, and quietly capable. Orbit should feel warm and personal without becoming playful or ornamental. It should carry the confidence and precision of a first-class macOS productivity application while preserving the recognizable character of the original web product.

## Anti-references

- Not a noisy project-management suite crowded with dashboards, configuration, or team features.
- Not a generic gray enterprise application or a cold blue-on-white SaaS template.
- Not a decorative canvas demo where visual effects obscure the underlying work.
- Not cloud-dependent, account-driven, telemetry-heavy, or subscription-shaped.
- Not an unfamiliar custom UI that discards standard macOS keyboard, focus, menu, and accessibility behavior.

## Design Principles

1. **The canvas is a working surface.** Spatial relationships, workflows, and annotations are core product capabilities, not decorative diagrams.
2. **Local ownership is visible.** Data operations must feel immediate, reliable, offline, and exportable.
3. **Keyboard paths are first-class.** Every frequent action should have a fast keyboard route, centered on the command palette.
4. **Calmness comes from hierarchy.** Keep screens spacious and restrained while preserving enough density for serious daily use.
5. **Native where it improves trust.** Match the web product's information architecture and visual identity, while using macOS conventions for windows, menus, focus, gestures, and file operations.

## Accessibility & Inclusion

Use native semantic controls wherever possible and support full keyboard operation, visible focus, VoiceOver labels, sufficient text and state contrast, reduced motion, scalable text, and color-independent state cues. Canvas nodes must remain real accessible views; only non-interactive edges, grids, and strokes should use immediate-mode drawing.
